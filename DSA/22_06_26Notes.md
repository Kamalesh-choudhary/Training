    10 20 30 40 50

- -10^5 ≤ Node value ≤ 10^5

```C++
        data = val;
        prev = NULL;
        next = NULL;
    }
};


//===== Declare Local Variables / Functions here if required =====

    tail->next = newnode;
    newnode->prev = tail;
    tail = tail->next;

    Node* head = NULL;
    Node* tail = NULL;

    for(int i=0;i<N;i++){
        int x;
```

### Q9] Delete the middle Node of a Linked list

The "middle" node is defined as the node at index floor(n / 2) using 0-based indexing, where n is the length of the list. For example, for the list [1,3,4,7,1,2,6] (n = 7) the middle is at index 3 (value 7), so deleting it results in [1,3,4,1,2,6]. For n = 1, removing the middle node results in an empty list. Typical approach: use slow and fast pointers to find the middle.

#### Output Format:

#### Constraints:
- 1 <= n <= 10^5
- Node values are integers that fit in a 32-bit signed integer.

```C++
    Node* tail = NULL;
```

    3
    10 20 30

    10 20 30

```C++
    Node* next;
```

### Q18] Middle of the Linked list
Given the head of a singly linked list, return the middle node of the linked list.

If there are two middle nodes, return the second middle node.
#### Input Format:
    head = [1,2,3,4,5]
#### Output Format: 
    [3,4,5]
#### Constraints:

    The number of nodes in the list is in the range [1, 100].

#### Code:
```C++
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode() : val(0), next(nullptr) {}
 *     ListNode(int x) : val(x), next(nullptr) {}
 *     ListNode(int x, ListNode *next) : val(x), next(next) {}
 * };
 */
class Solution {
public:
    ListNode* middleNode(ListNode* head) {
        ListNode* slow = head;
        ListNode* fast = head;
        while(fast != nullptr && fast->next != nullptr){
            slow = slow->next;
            fast = fast->next->next;
        }
        return slow;
    }
```

Given head, the head of a linked list, determine if the linked list has a cycle in it.

There is a cycle in a linked list if there is some node in the list that can be reached again by continuously following the next pointer. Internally, pos is used to denote the index of the node that tail's next pointer is connected to. Note that pos is not passed as a parameter.
